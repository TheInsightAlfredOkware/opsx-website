Section 2

Phase 1 — Build the website locally

Setting up a React + Vite + TypeScript + Tailwind CSS project

Step 1: Create the project folder

Open a new terminal in VS Code and run:

# Navigate to where you want the project
cd C:\Users\yourname

# Create project folder manually (Vite interactive prompts can be unreliable)
mkdir my-website
cd my-website

Step 2: Create package.json

Create a file called package.json in the project root with your dependencies listed. Key dependencies for a React + Vite + Tailwind site:

"dependencies": {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "react-router-dom": "^6.22.0",
  "framer-motion": "^11.0.0",
  "lucide-react": "^0.344.0",
  "react-helmet-async": "^2.0.5",
  "@formspree/react": "^2.5.1"
},
"devDependencies": {
  "vite": "^5.4.0",
  "@vitejs/plugin-react": "^4.3.1",
  "typescript": "^5.4.0",
  "tailwindcss": "^3.4.0",
  "postcss": "^8.4.0",
  "autoprefixer": "^10.4.0"
}

Step 3: Install dependencies

npm install

This creates a node_modules/ folder and package-lock.json. You should see "added X packages" with 0 critical vulnerabilities.

Step 4: Configure Vite with a fixed port

In vite.config.ts, always set a fixed port to prevent conflicts with other running projects:

server: {
  port: 5200,
  strictPort: true, // fails instead of silently stealing another port
},

Step 5: Run the development server

npm run dev

Open your browser at the URL shown (e.g. http://localhost:5200/). This is your live preview while building.

Important: Always open a NEW terminal window when starting a second project's dev server. Never run two projects from the same terminal. Use strictPort: true so if the port is taken, it stops rather than stealing another project's port.

Step 6: Check for TypeScript errors

Before building for production, always run a type check:

npx tsc --noEmit

This must return with no output (no errors). Common issue: single-quoted strings containing apostrophes (e.g. 'what's working') will break. Fix by using double quotes ("what's working") or backtick template literals.

Section 3

Phase 2 — Set up Formspree (contact form)

Free contact form backend — no server required
1. Go to formspree.io → sign up for a free account
2. Click "+ New Form" → give it a name (e.g. "Ops X Contact Form") → Create
3. Copy the Form ID shown — it looks like xabcdefg
4. Create a file called .env.local in your project root (never commit this file):
VITE_FORMSPREE_ID=xabcdefg
5. In your contact form component, use: useForm("xabcdefg") from @formspree/react
Free tier limits: Formspree free allows 50 submissions/month. This is plenty for a new business. Upgrade when needed.
Security note: Never put your Formspree ID or any secret keys directly in your code files. Always use environment variables (import.meta.env.VITE_*) and add .env.local to your .gitignore.
Section 4

Phase 3 — Build for production

Creating the optimised files that get deployed

Standard build

cd C:\Users\yourname\my-website
npm run build

This runs tsc && vite build and creates a dist/ folder. The output looks like:

dist/index.html             2.56 kB
dist/assets/vendor-xxx.js   163 kB
dist/assets/motion-xxx.js   115 kB
dist/assets/index-xxx.css    30 kB
✓ built in 3s
The dist/ folder is what you deploy. Every time you make changes and want to update your live site, run npm run build again and redeploy the new dist/ folder.

Build verification checklist before deploying

  • ✓ npx tsc --noEmit returns zero errors
  • ✓ npm run build exits with code 0 (no errors)
  • ✓ dist/ folder exists and contains index.html
  • ✓ Test locally with npm run preview to confirm the build works
Section 5

Phase 4 — Deploy to Netlify

Getting your site live on the internet for free

Method A: Drag and Drop (simplest)

1. Go to app.netlify.com → sign up / log in (use GitHub for easier future integration)
2. From the dashboard, click "Add new site" → "Deploy manually"
3. Open File Explorer and navigate to your project folder
4. Drag the entire dist/ folder and drop it onto the Netlify upload zone
5. Netlify will deploy instantly and give you a URL like https://random-name.netlify.app
6. To change the site name: Site configuration → Change site name → type your preferred slug (e.g. opsxconsulting) → your URL becomes https://opsxconsulting.netlify.app

Add environment variables to Netlify

Your site needs the Formspree ID to work in production. Add it in Netlify:

1. Go to your site in Netlify → Site configuration → Environment variables
2. Click "Add a variable"
3. Key: VITE_FORMSPREE_ID   Value: your Formspree form ID
4. Save → then rebuild locally (npm run build) and redeploy the new dist/
Why rebuild after adding env vars? Vite bakes environment variables into the JavaScript bundle at build time. If you add the variable in Netlify but don't rebuild, the old bundle still has no value for it.

Redeploying after changes

npm run build

Then drag the new dist/ folder to Netlify → Deploys tab → drop zone. Takes about 30 seconds.

Section 7

Phase 6 — Connect the domain to Netlify

Telling Netlify about your domain
1. In Netlify, go to your site → Domain management (left sidebar)
2. Click "Add a domain"
3. Type your domain (e.g. opsxafrica.com) → click Verify
4. Netlify confirms it found the domain → click "Add domain"
5. You'll see your domain listed as "Primary domain" with a yellow "Pending DNS verification" warning — this is expected
6. Click "Pending DNS verification" → click "Set up Netlify DNS for [yourdomain.com] →" at the bottom
7. Click Verify — Netlify will display 4 nameserver addresses. Copy all 4.

The nameservers will look like this:

dns1.p05.nsone.net
dns2.p05.nsone.net
dns3.p05.nsone.net
dns4.p05.nsone.net
Keep this Netlify page open. You'll need these nameservers in the next step. The exact values (p01, p02, p03, p05 etc.) vary per account — use whatever Netlify shows you, not the example above.
Section 8

Phase 7 — Point Namecheap nameservers to Netlify

Telling Namecheap to hand DNS control to Netlify
1. Go to namecheap.com → log in → click Domain List in the left sidebar
2. Find your domain → click Manage
3. You are on the Domain tab — scroll down to the NAMESERVERS section
4. Click the dropdown showing "Namecheap BasicDNS" → change it to "Custom DNS"
5. Four input fields appear — enter the 4 nameservers from Netlify exactly as shown
6. Click the green checkmark (✓) to save
7. A confirmation dialog may appear — confirm the change
What this does: You are transferring DNS control from Namecheap to Netlify. Netlify will now manage all DNS records for your domain — including automatically creating the records to point to your site and issuing SSL certificates.
Propagation time: After saving, DNS changes take 5 minutes to 24 hours to propagate globally. During this time, some visitors may still see the old state. This is normal. Average wait time is 15–30 minutes.
Section 9

Phase 8 — Provision the SSL certificate (HTTPS)

Getting the padlock and removing the "Not secure" warning

After setting the nameservers, Netlify needs to verify DNS is pointing correctly before it can issue the SSL certificate.

What you may see immediately after saving nameservers

If you visit your domain in a browser too early, you may see a "Your connection isn't private" / NET::ERR_CERT_COMMON_NAME_INVALID error. This is normal — it means DNS propagated but SSL is not yet issued. Do not be alarmed.

How to trigger SSL provisioning

1. Wait 15–30 minutes after saving nameservers in Namecheap
2. Go to Netlify → your site → Domain management → scroll down to the HTTPS section
3. Click "Verify DNS configuration"
4. If it shows "DNS verification was successful ✓" — Netlify is now provisioning your certificate automatically
5. Wait 5–10 minutes → visit https://yourdomain.com → you should see the padlock (🔒)
Success state: When SSL is active, Netlify's HTTPS section will show "Your site has HTTPS enabled" with a green indicator. Your domain is now fully live and secure.
If "Verify DNS configuration" keeps failing: Wait longer — up to 24 hours in rare cases. Do not click "Provide your own certificate" — that is for advanced users with their own SSL certificates. Netlify's automatic free certificate (Let's Encrypt) is the correct option.
Section 10

Phase 9 — Add a subdomain for a second app

e.g. app.opsxafrica.com → Vercel (ITS V.Pro app)

If you have a second application (like ITS V.Pro on Vercel) that you want to run under the same domain as app.yourdomain.com:

Step 1: Add subdomain in Vercel

1. Go to vercel.com → open your project
2. Go to Settings → Domains
3. Type app.yourdomain.com → click Add
4. Vercel will show you a CNAME value to add — it looks like cname.vercel-dns.com

Step 2: Add CNAME record in Netlify DNS

Since Netlify is now managing your DNS (you pointed nameservers to Netlify in Phase 7), you add the Vercel CNAME record through Netlify's DNS panel:

1. Go to Netlify → Domains (top menu, not inside a specific site)
2. Click on yourdomain.com
3. Click "Add new record"
4. Type: CNAME  |  Name: app  |  Value: cname.vercel-dns.com (or whatever Vercel showed you)
5. Save → wait 5–15 minutes
6. Back in Vercel, click "Verify" on the domain — it should confirm and provision SSL automatically

Section 12

Troubleshooting common issues

Quick fixes for the most common problems
ProblemCauseFix
npm run dev steals another project's port No fixed port set Add server: { port: 5200, strictPort: true } to vite.config.ts
TypeScript error: ',' expected Apostrophe inside single-quoted string (e.g. 'it's') Change to double quotes: "it's"
Contact form not delivering emails VITE_FORMSPREE_ID not set in Netlify Add env var in Netlify → rebuild → redeploy
"Your connection isn't private" on custom domain DNS propagated but SSL not yet issued Wait 10–30 min → go to Netlify HTTPS section → click "Verify DNS configuration"
Domain still shows Namecheap parking page DNS not yet propagated Wait up to 24 hours. Check progress at dnschecker.org
Page shows fine on home route but 404 on other routes React Router needs SPA redirect rule Create public/_redirects file with content: /* /index.html 200
import.meta.env TypeScript error Missing Vite client types Add "types": ["vite/client"] to tsconfig.json compilerOptions
Netlify SPA redirect fix: Create a file at public/_redirects (no extension) with exactly this content: /* /index.html 200. This tells Netlify to serve your React app for all routes. Without this, refreshing a page like /about will return a 404.
